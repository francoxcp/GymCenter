2. Configuración de credenciales (obligatorio)

Las credenciales de Supabase no están dentro del código fuente (seguridad). Se inyectan en tiempo de compilación. Sin este paso la app no se conecta al backend.

2.1 Crear archivo de credenciales locales

1Ve a la carpeta scripts/ del proyecto
2Copia local_credentials.bat.example y renómbralo local_credentials.bat
3Abre el archivo con el Bloc de Notas y rellena:
set SUPABASE_URL=https://xxxxxxxxxxxx.supabase.co
set SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxxx

2.2 Dónde encontrar las credenciales

  1. Ve a supabase.com e inicia sesión
  2. Selecciona tu proyecto Chamos Fitness Center
  3. Menú izquierdo: Settings → API
  4. Copia Project URL → es el SUPABASE_URL
  5. Copia anon (public) key → es el SUPABASE_ANON_KEY
⚠️ Importante: Nunca uses la clave service_role en la app móvil. Usa solo la anon.

4. Política de privacidad pública

Ambas tiendas exigen una URL pública. El archivo ya existe en web_deploy/privacy.html.

Opción A — GitHub Pages (gratis, 10 min)

  1. Crea cuenta en github.com
  2. Crea repositorio nuevo llamado chamos-legal
  3. Sube web_deploy/privacy.html como index.html
  4. En el repo: Settings → Pages → Source: main → Save
  5. Tu URL: https://TU_USUARIO.github.io/chamos-legal/

Opción B — Netlify (gratis, 5 min)

  1. Ve a netlify.com
  2. Arrastra la carpeta web_deploy/ al área de drop
  3. Obtienes URL tipo https://nombre.netlify.app
ℹ️ Guarda esta URL — la necesitarás al completar el listing en ambas tiendas.

5. ANDROID — Google Play Store

5.1 Crear cuenta de desarrollador (pago único)

  1. Ve a play.google.com/console
  2. Inicia sesión con cuenta Google del negocio
  3. Paga el registro: US$25 pago único
  4. Completa el perfil: nombre, dirección, teléfono
  5. La verificación tarda 24–48 horas

5.2 Crear el Keystore de firma (una sola vez en la vida del proyecto)

⚠️ CRÍTICO: Si pierdes este archivo, no puedes publicar actualizaciones jamás. Guárdalo en Google Drive, USB y otro lugar seguro.

Abre PowerShell y ejecuta:

keytool -genkey -v ^
  -keystore chamos-release.jks ^
  -keyalg RSA -keysize 2048 -validity 10000 ^
  -alias chamos

El comando pedirá:

  • Contraseña del keystore: elige una segura y anótala
  • Nombre y apellido: nombre de la empresa
  • Organización: Chamos Fitness Center
  • Ciudad / Estado / País: los tuyos (ej: Caracas / Miranda / VE)
  • Contraseña del alias: puede ser la misma que la del keystore

Mueve el archivo generado:

move chamos-release.jks d:\ChamosFitnessCenter\android\app\chamos-release.jks

5.3 Crear android/key.properties

Crea el archivo android/key.properties (ya está en .gitignore) con:

storePassword=LA_CONTRASEÑA_DEL_KEYSTORE
keyPassword=LA_CONTRASEÑA_DEL_ALIAS
keyAlias=chamos
storeFile=chamos-release.jks

5.4 Compilar el App Bundle (AAB)

ℹ️ Google Play solo acepta formato AAB desde 2021. No subas APK sueltos.
flutter build appbundle --release ^
  --dart-define=SUPABASE_URL=https://tu-proyecto.supabase.co ^
  --dart-define=SUPABASE_ANON_KEY=tu-anon-key ^
  --obfuscate ^
  --split-debug-info=build/debug-info

O usa el script automático (selecciona opción 2):

cd d:\ChamosFitnessCenter\scripts
build_production.bat

Archivo generado en: build\app\outputs\bundle\release\app-release.aab

5.5 Crear la app en Play Console

  1. Clic en "Crear app"
  2. Nombre: Chamos Fitness Center
  3. Idioma: Español (Latinoamérica)
  4. Tipo: App | Gratis
  5. Acepta las declaraciones → "Crear app"

5.7 Configurar contenido de la app

  • Calificación de contenido: responde el cuestionario (probablemente PEGI 3 / Everyone)
  • Público objetivo: 18+
  • Acceso a la app: crea usuario de prueba y proporciona credenciales a los revisores

  • Seguridad de los datos: declara: nombre, email, fotos de perfil, datos de fitness

5.8 Subir el AAB y enviar a revisión

  1. Ve a Versiones de la app → Producción → Crear nueva versión
  2. Acepta Play App Signing cuando lo ofrezca (recomendado)
  3. Sube el archivo app-release.aab
  4. Novedades: "Versión inicial de Chamos Fitness Center"
  5. Clic en Guardar → Revisar versión → Enviar a producción
ℹ️ La revisión tarda 3–7 días hábiles la primera vez. Recibirás email con el resultado.

6. iOS — Apple App Store

⚠️ Requiere Mac con Xcode. No se puede compilar para iOS desde Windows. Sin Mac, usa Codemagic (codemagic.io) que tiene plan gratuito para Flutter.

6.1 Crear cuenta Apple Developer (renovación anual)

  1. Ve a developer.apple.com/programs/enroll
  2. Inicia sesión con Apple ID (o crea uno)
  3. Selecciona "Individual" o "Organization"
  4. Paga: US$99 al año
  5. La verificación tarda 24–48 horas

6.2 Registrar el Bundle Identifier

  1. Ve a developer.apple.com/account
  2. Certificates, Identifiers & Profiles → Identifiers → (+)
  3. Selecciona "App IDs" → "App"
  4. Bundle ID: com.chamosfitness.app (modo Explicit)
  5. Registrar

6.3 Crear certificado de distribución

  1. En Certificates → (+) selecciona "Apple Distribution"
  2. En tu Mac: abre Keychain Access → Certificate Assistant → Request a Certificate from a CA
  3. Sube el CSR generado y descarga el .cer
  4. Doble clic en el .cer para instalarlo en tu Mac

6.4 Crear Provisioning Profile

  1. En Profiles → (+) selecciona "App Store Connect"
  2. Selecciona App ID: com.chamosfitness.app
  3. Selecciona tu certificado de distribución
  4. Nombre: ChamosFitness AppStore
  5. Descarga e instala el .mobileprovision (doble clic)

6.5 Crear la app en App Store Connect

  1. Ve a appstoreconnect.apple.com
  2. Mis apps → (+) → Nueva app
  3. Plataformas: iOS | Nombre: Chamos Fitness Center
  4. Bundle ID: selecciona com.chamosfitness.app
  5. SKU: chamosfitnesscenter

6.8 TestFlight — pruebas antes de publicar

  1. El build aparece en TestFlight procesándose (~10 min)
  2. Agrega testers internos (máx 100 personas)
  3. Prueba la app durante unos días antes de publicar

6.9 Enviar a revisión de App Store

  1. En App Store Connect → selecciona el build de TestFlight
  2. Completa "Sign-In Information": usuario y contraseña de prueba para el revisor
  3. Clic en "Agregar a revisión" → "Enviar para revisión"
ℹ️ La revisión tarda 1–3 días hábiles. Actualizaciones posteriores son más rápidas.

7. Cómo publicar actualizaciones

Paso 1 — Subir la versión en pubspec.yaml

# Antes
version: 1.0.0+1

# Después (ejemplo)
version: 1.0.1+2

El número antes del + es la versión visible al usuario. El número después del + es el código interno — debe subir siempre y nunca repetirse.

Paso 2 — Compilar

# Android
flutter build appbundle --release ^
  --dart-define=SUPABASE_URL=... ^
  --dart-define=SUPABASE_ANON_KEY=...

# iOS (desde Mac)
flutter build ipa --release \
  --dart-define=SUPABASE_URL=... \
  --dart-define=SUPABASE_ANON_KEY=...

Paso 3 — Subir

  • Android: Play Console → Producción → Crear nueva versión → subir nuevo .aab
  • iOS: App Store Connect → nueva versión → subir build desde Xcode

Paso 4 — Revisión

  • Google: 1–3 días (actualizaciones más rápidas que la primera publicación)
  • Apple: 1–2 días

8. Solución de problemas frecuentes

Android

ErrorSolución
keystore file not foundVerifica que android/app/chamos-release.jks existe y que storeFile en key.properties es correcto
SUPABASE_URL is emptyEl build se hizo sin --dart-define. Incluye ambas variables en el comando
"You uploaded an APK..."Usa flutter build appbundle, no flutter build apk
versionCode already usedIncrementa el número después del + en pubspec.yaml

iOS

ErrorSolución
No signing certificateEl certificado de distribución no está instalado. Descárgalo de developer.apple.com y haz doble clic
Provisioning profile not foundVerifica que el Bundle ID en Xcode es exactamente com.chamosfitness.app
Rechazo: "Missing privacy usage"Agrega la descripción del permiso en ios/Runner/Info.plist
Rechazo: "App uses encryption"Declara uso de cifrado estándar (HTTPS/TLS) en App Store Connect y marca la exención
